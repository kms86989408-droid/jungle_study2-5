핵심 요청:

## 2. 다룬 코드

사용한 코드는 아래와 같습니다.

```python
import sys
input = sys.stdin.readline

n = int(input())
nums = list(map(int, input().split()))

max_sum = 0
visited = [False] * n
selected = []

def solve():
    global max_sum

    if len(selected) == n:
        current_sum = 0

        for i in range(n - 1):
            current_sum += abs(selected[i] - selected[i + 1])

        if current_sum > max_sum:
            max_sum = current_sum
        return

    for i in range(n):
        if not visited[i]:
            visited[i] = True
            selected.append(nums[i])

            solve()

            selected.pop()
            visited[i] = False

solve()
print(max_sum)
```

---

## 3. 이 코드의 목표

이 코드는 숫자들의 순서를 바꿔서 모든 경우를 만들어 보고,

```python
|A[0]-A[1]| + |A[1]-A[2]| + ... + |A[n-2]-A[n-1]|
```

값이 **가장 크게 되는 경우의 합**을 찾는 코드입니다.

즉,
- 숫자를 한 개씩 골라 순서를 만들고
- 끝까지 다 고르면 값을 계산하고
- 최댓값을 저장하고
- 다시 돌아와 다른 경우를 시도합니다.

이 과정이 바로 **백트래킹(Backtracking)** 입니다.

---

## 4. 가장 먼저 수정해야 했던 부분

원래 코드에는 아래 줄이 있었습니다.

```python
nums = map(int, input().split())
```

이렇게 쓰면 `nums[i]` 처럼 인덱스로 접근할 수 없습니다.

그래서 아래처럼 바꿔야 합니다.

```python
nums = list(map(int, input().split()))
```

이유:
- `map(...)`은 바로 인덱싱이 안 되고
- 뒤에서 `selected.append(nums[i])`를 사용하고 있기 때문입니다.

## 5. 주요 변수 역할 정리

### `n`
숫자의 개수입니다.

영향 가는 곳:
- `visited = [False] * n`
- `if len(selected) == n`
- `for i in range(n)`
- `for i in range(n - 1)`

즉, **반복 횟수와 종료 조건의 기준**입니다.

---

### `nums`
입력받은 숫자 리스트입니다.

예:

```python
nums = [20, 1, 15, 8, 4, 10]
```

영향 가는 곳:
- `selected.append(nums[i])`

즉, 실제로 선택되는 숫자들입니다.

---

### `max_sum`
지금까지 찾은 최댓값을 저장하는 변수입니다.

영향 가는 곳:

```python
if current_sum > max_sum:
    max_sum = current_sum
```

마지막에 `print(max_sum)`으로 출력됩니다.

---

### `visited`
각 숫자를 이미 사용했는지 확인하는 체크 배열입니다.

예:

```python
visited = [False, False, False, False, False, False]
```

- `False` → 아직 안 씀
- `True` → 이미 씀

영향 가는 곳:

```python
if not visited[i]:
```

즉, **중복 선택을 막는 장부**입니다.

---

### `selected`
현재까지 고른 숫자들을 순서대로 저장하는 리스트입니다.

예:

```python
selected = [20, 1, 15]
```

영향 가는 곳:
- `if len(selected) == n`
- `selected[i]`, `selected[i+1]`
- `selected.append(...)`
- `selected.pop()`

즉, **지금 만들고 있는 순열 자체**입니다.

---

## 6. `solve()` 함수의 흐름

### 1) 종료 조건

```python
if len(selected) == n:
```

의미:
- 숫자를 전부 골랐다면 더 내려가지 않고 계산을 시작합니다.

---

### 2) 현재 순열 값 계산

```python
current_sum = 0
for i in range(n - 1):
    current_sum += abs(selected[i] - selected[i + 1])
```

예:

```python
selected = [20, 1, 15, 8, 4, 10]
```

이면

```text
|20-1| + |1-15| + |15-8| + |8-4| + |4-10|
= 19 + 14 + 7 + 4 + 6
= 50
```

---

### 3) 최댓값 갱신

```python
if current_sum > max_sum:
    max_sum = current_sum
```

이번 순열의 결과가 지금까지 최고값보다 크면 갱신합니다.

---

### 4) 다음 숫자 선택

```python
for i in range(n):
    if not visited[i]:
```

아직 사용하지 않은 숫자만 다음 자리에 넣습니다.

---

### 5) 백트래킹 핵심 동작

```python
visited[i] = True
selected.append(nums[i])

solve()

selected.pop()
visited[i] = False
```

사람 말로 바꾸면:
1. 이 숫자를 사용했다고 표시하고
2. 바구니에 넣고
3. 다음 자리로 내려가 보고
4. 다 보고 오면 다시 빼고
5. 사용 표시도 원래대로 돌립니다.

이 패턴이 바로 **선택 → 재귀 호출 → 취소** 입니다.

---

## 7. 작은 예시 트리: `nums = [1, 2, 3]`

```text
solve() 시작
selected = []

├── 1 선택
│   ├── 2 선택
│   │   └── 3 선택
│   │       → [1, 2, 3]
│   │       → |1-2| + |2-3| = 2
│   └── 3 선택
│       └── 2 선택
│           → [1, 3, 2]
│           → |1-3| + |3-2| = 3
│
├── 2 선택
│   ├── 1 선택
│   │   └── 3 선택
│   │       → [2, 1, 3]
│   │       → 3
│   └── 3 선택
│       └── 1 선택
│           → [2, 3, 1]
│           → 3
│
└── 3 선택
    ├── 1 선택
    │   └── 2 선택
    │       → [3, 1, 2]
    │       → 3
    └── 2 선택
        └── 1 선택
            → [3, 2, 1]
            → 2
```

리프 노드(맨 끝)까지 가면 하나의 순열이 완성되고, 그때 계산합니다.

## 8. 큰 예시 트리: `n = 6`, `nums = [20, 1, 15, 8, 4, 10]`

전체 경우의 수는 다음과 같습니다.

```text
6! = 720
```

그래서 전체 트리를 모두 그리기보다는 구조를 이해하는 것이 중요합니다.

### 전체 트리의 큰 모양

```text
[]
├── [20]
├── [1]
├── [15]
├── [8]
├── [4]
└── [10]
```

예를 들어 `[20]` 아래에는 남은 5개 숫자가 붙습니다.

```text
[20]
├── [20, 1]
├── [20, 15]
├── [20, 8]
├── [20, 4]
└── [20, 10]
```

`[20, 1]` 아래에는 남은 4개 숫자가 붙습니다.

```text
[20, 1]
├── [20, 1, 15]
├── [20, 1, 8]
├── [20, 1, 4]
└── [20, 1, 10]
```

이런 식으로 끝까지 내려갑니다.

---

## 9. 한 갈래를 끝까지 따라가 보기

예를 들어 다음 경로를 따라 내려간다고 해봅시다.

```text
[]
→ [20]
→ [20, 1]
→ [20, 1, 15]
→ [20, 1, 15, 8]
→ [20, 1, 15, 8, 4]
→ [20, 1, 15, 8, 4, 10]
```

이제 길이가 6이므로 종료 조건을 만족합니다.

```python
if len(selected) == n:
```

이때 계산값은:

```text
|20-1| + |1-15| + |15-8| + |8-4| + |4-10|
= 19 + 14 + 7 + 4 + 6
= 50
```

---

## 10. 백트래킹으로 다시 올라오는 모습

`[20, 1, 15, 8, 4, 10]` 계산이 끝나면 함수는 돌아오고,

```python
selected.pop()
visited[i] = False
```

가 실행됩니다.

즉,

```text
[20, 1, 15, 8, 4, 10]
↩ [20, 1, 15, 8, 4]
↩ [20, 1, 15, 8]
→ [20, 1, 15, 8, 10]
→ [20, 1, 15, 8, 10, 4]
```

처럼 한 갈래를 다 보면 다시 올라와서 **다른 가지를 탐색**합니다.

이게 백트래킹의 핵심입니다.

---

## 11. 깊이별 가지 수

`n = 6`일 때 각 깊이에서 가능한 경우 수는 다음과 같습니다.

- 깊이 0: `1개` (`[]`)
- 깊이 1: `6개`
- 깊이 2: `6 × 5 = 30개`
- 깊이 3: `6 × 5 × 4 = 120개`
- 깊이 4: `6 × 5 × 4 × 3 = 360개`
- 깊이 5: `6 × 5 × 4 × 3 × 2 = 720개`
- 깊이 6: `6! = 720개` 완성

즉, 마지막에는 총 **720개의 순열**을 모두 검사합니다.

## 12. 이 코드에서 꼭 기억해야 할 핵심

### 1) `selected`
현재까지 만든 경로입니다.

예:

```python
[20, 1, 15]
```

---

### 2) `visited`
이미 쓴 숫자를 다시 고르지 못하게 막습니다.

그래서 한 경로에 같은 숫자가 두 번 나오지 않습니다.

---

### 3) 리프 노드에서 계산
길이가 `n`이 되면 하나의 순열이 완성된 것이므로 점수를 계산합니다.

---

### 4) 백트래킹의 본질
아래 4줄이 핵심입니다.

```python
visited[i] = True
selected.append(nums[i])
solve()
selected.pop()
visited[i] = False
```

의미:
- 선택하고
- 내려가 보고
- 다시 취소하고
- 다른 경우를 시도한다
